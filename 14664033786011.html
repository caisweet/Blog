properties简例

2016/6/20 posted in  Java

2016年06月20日

问题

判断运行的次数，如果超过5次就中断退出

思路

  1. 要判断次数必须有一个计数器
  2. 因为要计算运行次数的，所以计数器不能在内存中，否则每次都重置了
  3. 可以使用properties配合输入流和输出流，通过读写文件来判断次数，从而中断程序
st=>start: 开始
e=>end: 结束
op=>operation: 读取
opt=>operation: 创建配置文件
stop=>operation: 中断程序
continue=>operation: 计数器+1，程序正常运行
cond=>condition: 判断是否存在配置文件？
if=>condition: 判断次数是否大于5?
st->cond(yes)->op->if(yes)->stop->e
cond(no)->opt->op
if(no)->continue->e

代码

import java.io.File;
import java.io.FileInputStream;
import java.io.FileOutputStream;
import java.io.IOException;
import java.util.Properties;

public class Java20160620_1 {
    public static void main(String[]args) throws IOException {
        File file = new File("time.properties");
        //判断配置文件是否存在，不存在就建立一个
        if (!file.exists()){
            file.createNewFile();
        }
        FileInputStream fis = new FileInputStream(file);
        int count=0;
        Properties prop=new Properties();
        prop.load(fis);
        String value=prop.getProperty("time");
        //判断配置文件中是否存在time相对于值，存在就读取
        if (value!=null){
            count=Integer.parseInt(value);
            //如果次数大于5，就抛出运行时异常并中断程序
            if (count>=5){
                throw new RuntimeException("运行次数已到限制!");
            }
        }
        count++;
        System.out.println("这是软件第"+count+"次运行");
        prop.setProperty("time",count+"");
        FileOutputStream fos=new FileOutputStream(file);
        //将次数写进配置文件
        prop.store(fos,"");
        fis.close();
        fos.close();
    }
}

总结

因为只是一个数字的读取，所以文件格式没整理。如果数据一多，xml文件的好处就出来了，就当做是管中窥豹吧。
